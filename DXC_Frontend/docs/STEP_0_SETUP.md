# STEP 0: Setup API & Feature Structure

## Overview

Setup foundation for a new feature:
1. Generate API types using Orval
2. Create API call layer
3. Create data hooks (TanStack Query)
4. Setup feature folder structure
5. Create type definitions

**Estimated time**: 20-30 minutes

### Step 2: Understanding customRequest & API Response Normalization

**Important**: The project uses a `customRequest` function in `src/api/request.ts` to automatically normalize API responses and handle errors. This is critical for understanding how API integration works.

**File**: `src/api/request.ts` (already exists in project)

**How it works**:
```typescript
// src/api/request.ts handles:
export const customRequest = async <T>(config: AxiosRequestConfig): Promise<T> => {
  // 1. Automatically adds Bearer token to requests
  // 2. For ApiResult/PagedResult responses:
  //    - If success: true → returns response.data
  //    - If success: false → throws Error with response.message
  // 3. For network errors → throws Error with details
  
  // Result: Components NEVER need to check response.success
}
```

**Key Benefit**: All generated API clients automatically use this `customRequest`, so error handling is centralized and consistent.

### Step 3: Create API Call File

**Note**: With Orval, your API calls are **auto-generated** in `src/api/endpoints/[feature-name].ts`. You typically don't create these manually. They look like:

**Auto-generated File**: `src/api/endpoints/[feature-name].ts`

```typescript
// This is auto-generated by Orval - DO NOT EDIT MANUALLY
import type { 
  [Feature]Dto,
  [Feature]CreateRequest,
  PagedResult,
  ApiResult,
} from '@/api/models'

export const get[FeatureApi] = () => ({
  // LIST - Returns PagedResult<T>
  getApi[Features]: (params?: {
    Current?: number
    PageSize?: number
    Name?: string
  }) => customRequest<PagedResult<[Feature]Dto>>({
    url: '/api/v1/[feature-name]s',
    method: 'GET',
    params,
  }),

  // DETAIL - Returns ApiResult<T>
  getApi[Feature]: (id: string) => customRequest<ApiResult<[Feature]Dto>>({
    url: `/api/v1/[feature-name]s/${id}`,
    method: 'GET',
  }),

  // CREATE - Returns ApiResult<string> (new ID)
  postApi[FeatureCreate]: (data: [Feature]CreateRequest) => 
    customRequest<ApiResult<string>>({
    url: '/api/v1/[feature-name]s',
    method: 'POST',
    data,
  }),

  // UPDATE - Returns ApiResult<void>
  postApi[FeatureUpdate]: (data: [Feature]UpdateRequest) => 
    customRequest<ApiResult>({
    url: `/api/v1/[feature-name]s/${data.publicId}`,
    method: 'POST',
    data,
  }),

  // DELETE - Returns ApiResult<void>
  postApi[FeatureDelete]: (id: string) => 
    customRequest<ApiResult>({
    url: `/api/v1/[feature-name]s/${id}`,
    method: 'POST',
  }),
})
```

**Key Points About Response Types**:

| Operation | Response Type | What customRequest returns |
|-----------|---------------|---------------------------|
| **LIST** | `PagedResult<T>` | `{ data: T[], total, current, pageSize }` |
| **DETAIL** | `ApiResult<T>` | `{ data: T, message?: string }` |
| **CREATE** | `ApiResult<string>` | `{ data: "new-id", message?: string }` |
| **UPDATE** | `ApiResult<void>` | `{ success: true, message?: string }` |
| **DELETE** | `ApiResult<void>` | `{ success: true, message?: string }` |

**Error Handling**: When `success: false`, `customRequest` automatically throws an Error with the message from the API response.

**Where to find API definitions**:
- Backend API documentation (Swagger/OpenAPI)
- Generated types: `src/api/generated/` (auto-generated by Orval)
- Generated clients: `src/api/endpoints/` (auto-generated by Orval)
- Example: `src/api/endpoints/organizations.ts`

**Lưu ý**: Project sử dụng Vite Proxy nên không cần config `VITE_API_URL`. Tất cả API calls sẽ tự động được forward đến backend thông qua proxy configuration trong `vite.config.ts`.

### Step 4: Create Data Hooks

**File**: `src/features/[feature-name]/hooks/use[Features].ts`

Data hooks use TanStack Query for caching and state management. Note: Error handling is simplified because `customRequest` already throws errors automatically.

**Template**:
```typescript
import { useMutation, useQuery, useQueryClient } from '@tanstack/react-query'
import { toast } from 'sonner'
import { get[FeatureApi] } from '@/api/endpoints/[feature-name]'

const [featureApi] = get[FeatureApi]()

// ========================================
// QUERIES (Read-only)
// ========================================

// List items - Returns PagedResult<T>
export const use[Features] = (params: {
  Current?: number
  PageSize?: number
  Name?: string
}) => {
  return useQuery({
    queryKey: ['[features]', params],
    queryFn: () => featureApi.getApi[Features](params),
    staleTime: 5 * 60 * 1000, // 5 minutes
  })
}

// Single item detail - Returns ApiResult<T>
export const use[Feature]Detail = (id: string) => {
  return useQuery({
    queryKey: ['[feature]', id],
    queryFn: () => featureApi.getApi[Feature](id),
    enabled: !!id, // Only run if id exists
    staleTime: 5 * 60 * 1000,
  })
}

// ========================================
// MUTATIONS (Write operations)
// Note: customRequest already handles errors
// ========================================

// Create item - Returns ApiResult<string> (new ID)
export const useCreate[Feature] = () => {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: (data: any) => featureApi.postApi[FeatureCreate](data),
    onSuccess: () => {
      toast.success('Tạo thành công')
      // Invalidate list to refresh
      queryClient.invalidateQueries({ queryKey: ['[features]'] })
    },
    // onError is automatic - customRequest throws Error with message
    onError: (error: Error) => {
      toast.error(error.message)
    },
  })
}

// Update item - Returns ApiResult<void>
export const useUpdate[Feature] = () => {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: (data: any) => featureApi.postApi[FeatureUpdate](data),
    onSuccess: (_, variables) => {
      toast.success('Cập nhật thành công')
      // Invalidate list and detail
      queryClient.invalidateQueries({ queryKey: ['[features]'] })
      queryClient.invalidateQueries({ queryKey: ['[feature]', variables.publicId] })
    },
    onError: (error: Error) => {
      toast.error(error.message)
    },
  })
}

// Delete item - Returns ApiResult<void>
export const useDelete[Feature] = () => {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: (id: string) => featureApi.postApi[FeatureDelete](id),
    onSuccess: () => {
      toast.success('Xóa thành công')
      // Invalidate list
      queryClient.invalidateQueries({ queryKey: ['[features]'] })
    },
    onError: (error: Error) => {
      toast.error(error.message)
    },
  })
}
```

**Key Points**:
- `queryKey`: Identifies cache - must be consistent everywhere
- `staleTime`: How long before data is considered "stale" and needs refresh
- `enabled`: Conditionally enable query (only when id exists)
- `onSuccess`: Invalidate cache so UI updates
- `onError`: Just use `error.message` - `customRequest` already normalized it
- Always invalidate list after mutations
- **No need to check `response.success`** - `customRequest` handles this automatically

**Query Key Naming Convention**:
- List: `['[features]', params]` - includes params for different filters
- Detail: `['[feature]', id]` - includes id
- Pattern: `[resourceName, identifiers...]`

**Why Error Handling is Simple**:
When an API returns `{ success: false, message: "..." }`, `customRequest` automatically throws an `Error` with that message. So in `onError`, you just get the Error object and can use `error.message`.

### Real Example: Organization API

**File**: `src/api/organizations.ts`

```typescript
import { apiClient } from '@/api/client'
import type { 
  OrganizationDto,
  OrganizationCreateRequest,
  OrganizationUpdateRequest,
  PaginatedResponse,
} from '@/api/generated'

export const getOrganizations = async (params: {
  Current?: number
  PageSize?: number
  Name?: string
}) => {
  const response = await apiClient.get<PaginatedResponse<OrganizationDto>>(
    '/api/v1/organizations',
    { params }
  )
  return response.data
}

export const getOrganizationDetail = async (id: string) => {
  const response = await apiClient.get<OrganizationDto>(
    `/api/v1/organizations/${id}`
  )
  return response.data
}

export const createOrganization = async (data: OrganizationCreateRequest) => {
  const response = await apiClient.post<OrganizationDto>(
    '/api/v1/organizations',
    data
  )
  return response.data
}

export const updateOrganization = async (data: OrganizationUpdateRequest) => {
  const response = await apiClient.put<OrganizationDto>(
    `/api/v1/organizations/${data.publicId}`,
    data
  )
  return response.data
}

export const deleteOrganization = async (id: string) => {
  await apiClient.delete(`/api/v1/organizations/${id}`)
}
```

## Common Issues & Solutions

### Issue 1: "Cannot find module" error for API types

**Problem**: `[Feature]Dto` type doesn't exist in generated types

**Solution**:
1. Run `npm run gen:api` to regenerate
2. Check the generated type name matches what API returns
3. Look in `src/api/generated/` to see actual type names

### Issue 2: API call returns wrong data format

**Problem**: Response doesn't match typed structure

**Solution**:
1. Check backend API documentation
2. Verify correct endpoint path
3. Log API response: `console.log(response.data)`
4. Ask backend team if API schema changed

### Issue 3: Cache not updating after mutation

**Problem**: List doesn't refresh after create/update/delete

**Solution**:
- Make sure `onSuccess` calls `queryClient.invalidateQueries()`
- Verify queryKey matches exactly: `['[features]', params]`
- Check browser DevTools: React Query tab

### Issue 4: "queryClient is undefined"

**Problem**: Can't access queryClient in mutation

**Solution**:
```tsx
import { useQueryClient } from '@tanstack/react-query'

export const useCreateMyFeature = () => {
  const queryClient = useQueryClient()  // ← Add this
  return useMutation({
    // ...
  })
}
```

## Troubleshooting

### Build fails with types error

```
error TS7006: Parameter 'data' implicitly has an 'any' type.
```

**Fix**: Add type annotation:
```tsx
export const createMyFeature = async (data: MyFeatureCreateRequest) => {
  //                                        ↑ Add type
```

### Toast notifications don't appear

**Check**:
1. Is Sonner provider in layout? (check `src/app/layouts/RootLayout.tsx`)
2. Is import correct? `import { toast } from 'sonner'`
3. Typo in toast call? `toast.success()` not `toast.sucess()`

### Query not fetching data

**Debug**:
```tsx
const { data, isLoading, error } = use[Features]({...})

// Add logging
useEffect(() => {
  console.log('Query data:', data)
  console.log('Is loading:', isLoading)
  console.log('Error:', error)
}, [data, isLoading, error])
```

Check React Query DevTools in browser to see query status.
